QuickAPI (Chrome Extension)
Last updated: May 30, 2026

一、概述

「QuickAPI」（以下简称「本扩展」）是一款在浏览器本地运行的 HTTP 请求发送工具。我们非常重视您的隐私，本政策说明本扩展如何收集、使用和保护您的数据。

核心原则：本扩展所有数据均存储在您设备的本地浏览器中，不会将任何用户数据上传至任何远程服务器。

二、信息收集

2.1 本扩展收集的信息

数据类型来源用途
请求历史记录用户手动输入（URL、请求头、请求体）本地缓存，方便快速复用已发送的请求
界面语言偏好用户选择的语言选项记住您的语言选择，下次打开保持一致
临时编辑状态当前弹窗内的表单内容关闭/重新打开弹窗后恢复编辑进度

2.2 本扩展不收集的信息

  • ✗ 不收集任何个人身份信息（姓名、邮箱、电话等）
  • ✗ 不使用分析工具、追踪器或第三方统计代码
  • ✗ 不向任何服务器（包括开发者服务器）发送用户数据
  • ✗ 不读取您访问的网页内容（除用户主动输入的 URL 外）
  • ✗ 不包含任何广告或广告追踪代码

三、数据存储与安全

  • ✔ 所有数据通过 chrome.storage.local API 存储于您的设备本地
  • ✔ 数据不会离开您的浏览器，不经过任何网络传输
  • ✔ 卸载扩展后，所有本地存储数据将被自动清除
  • ✔ 用户可在扩展设置中随时清除全部存储数据

四、权限使用说明

本扩展申请了以下 Chrome 权限，具体用途如下：

权限用途
host_permissions: <all_urls>允许向用户指定的任意 URL 发送 HTTP 请求（GET/POST/PUT/DELETE/PATCH 等）。这是本扩展的核心功能，仅当用户主动点击「发送」按钮时才会发起网络请求。
activeTab获取当前标签页的 URL，用于自动填充到地址栏，方便用户操作。
storage在本地保存请求历史、语言偏好等用户设置。

五、HTTP 请求行为

  • HTTP 请求仅在用户主动点击「发送」按钮后发起
  • 目标 URL 由用户自行输入，扩展本身不会预设或推荐任何目标地址
  • 请求和响应内容不在本扩展的服务器中中转，直接由浏览器发送到目标服务器
  • 响应结果仅展示在弹窗内，不会被记录或外传

六、第三方服务

本扩展不依赖任何第三方在线服务。所有功能均在浏览器本地完成，无需连接外部服务器即可正常工作。

七、儿童隐私

本扩展不面向 13 岁以下儿童设计，也不会故意收集儿童的个人信息。

八、隐私政策更新

我们可能会不时更新本隐私政策。如有重大变更，我们将通过扩展商店页面发布通知。建议您定期查阅本页面的最新版本。本政策最后更新日期：2026 年 5 月 30 日。

1. Overview

"QuickAPI" (the "Extension") is an HTTP request sending tool that runs locally in your browser. We take your privacy seriously. This policy describes how the Extension collects, uses, and protects your data.

Core Principle: All data is stored locally on your device within the browser. No user data is ever uploaded to any remote server.

2. Information We Collect

2.1 Data Collected by This Extension

Data TypeSourcePurpose
Request historyManual user input (URL, headers, body)Local cache for quick reuse of previously sent requests
UI language preferenceUser's language selectionPersist language choice across sessions
Temporary edit stateCurrent form content in the popupRestore editing progress when reopening the popup

2.2 Data NOT Collected

  • ✗ No personally identifiable information (name, email, phone, etc.)
  • ✗ No analytics, trackers, or third-party statistics
  • ✗ No data sent to any server (including developer servers)
  • ✗ No reading of webpage content beyond what you manually enter as URLs
  • ✗ No ads or ad tracking code

3. Data Storage & Security

  • ✔ All data is stored via chrome.storage.local on your local device
  • ✔ Data never leaves your browser; no network transmission involved
  • ✔ All stored data is automatically deleted when you uninstall the Extension
  • ✔ Users can clear all stored data at any time via extension settings

4. Permissions Used

The Extension requests the following Chrome permissions:

PermissionPurpose
host_permissions: <all_urls>Allows sending HTTP requests (GET/POST/PUT/DELETE/PATCH) to any URL specified by the user. This is the core functionality of the Extension. Requests are only initiated when the user actively clicks "Send".
activeTabRetrieves the current tab's URL for convenient auto-filling in the address bar.
storageSaves request history, language preferences, and other settings locally.

5. HTTP Request Behavior

  • HTTP requests are sent only when the user actively clicks the "Send" button
  • The target URL is entered manually by the user; the Extension does not pre-configure or suggest any destination
  • Requests and responses are not proxied through any server owned by the developer; they go directly from the browser to the target server
  • Response results are displayed only in the popup and are never logged or transmitted externally

6. Third-Party Services

This Extension does not rely on any third-party online services. All functions run entirely within your browser with no need to connect to external servers.

7. Children's Privacy

This Extension is not directed at children under 13 and does not knowingly collect personal information from children.

8. Changes to This Policy

We may update this Privacy Policy from time to time. For material changes, we will provide notice through the extension store listing. Last updated: May 30, 2026.
